# Exercise 1 - Securing RESTful Web Services Using Spring Security

## Objective

Implement Spring Security in a Spring Boot RESTful Web Service to secure all REST endpoints using HTTP Basic Authentication.

---

## Technologies Used

- Java 17
- Spring Boot 3.5.16
- Spring Security
- Spring Web
- Maven
- Postman
- Git Bash
- Eclipse / STS

---

## Features

- Secure REST APIs using Spring Security
- Enable HTTP Basic Authentication
- Restrict access to all REST endpoints
- Authenticate users with the default Spring Security user
- Access secured APIs using Browser, Postman and Curl
- Return HTTP 401 Unauthorized for unauthenticated requests
- Return HTTP 200 OK for authenticated requests

---

## Dependency Added

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

---

## Security Configuration

A new package **security** is created containing the class:

```
SecurityConfig.java
```

The class is annotated with:

- `@Configuration`
- `@EnableWebSecurity`

A `SecurityFilterChain` bean is configured to:

- Authenticate all requests
- Enable HTTP Basic Authentication
- Disable CSRF protection for REST API testing

---

## REST Endpoint

### Get All Countries

**Method**

```
GET
```

**URL**

```
http://localhost:8083/countries
```

---

## Testing Without Authentication

### Request

```
GET http://localhost:8083/countries
```

### Response

```
HTTP/1.1 401 Unauthorized
```

This indicates that authentication is required to access the secured REST endpoint.

---

## Testing With Authentication

### Username

```
user
```

### Password

```
Generated by Spring Security at application startup.
```

### Curl Command

```bash
curl -i -u user:<generated-password> http://localhost:8083/countries
```

### Response

```
HTTP/1.1 200 OK
```

```json
[
    {
        "code":"IN",
        "name":"India"
    },
    {
        "code":"US",
        "name":"United States"
    },
    {
        "code":"JP",
        "name":"Japan"
    },
    {
        "code":"DE",
        "name":"Germany"
    }
]
```

## Postman Testing

- Method : GET
- URL : `http://localhost:8083/countries`
- Authorization Type : Basic Auth
- Username : `user`
- Password : Generated Spring Security Password

Successful authentication returns the list of countries in JSON format.

## How to Run

1. Import the project into Eclipse/STS.
2. Update Maven dependencies.
3. Run `SpringLearnApplication.java`.
4. Note the generated password displayed in the console.
5. Access the REST API using:
   - Browser
   - Postman
   - Git Bash (Curl)
6. Verify:
   - Without authentication → HTTP 401 Unauthorized
   - With authentication → HTTP 200 OK

---

## Output

Successfully secured the RESTful Web Service using Spring Security. All REST endpoints require authentication using HTTP Basic Authentication. Authorized users receive the requested JSON response, while unauthorized users receive HTTP 401 Unauthorized.
